Visão Geral e Funcionalidades do Painel

Bem-vindo ao painel do Neuralogins! Aqui você pode gerenciar a integração do nosso sistema de login nas suas próprias aplicações. As principais funcionalidades disponíveis para você são:

  • Gerenciar Chaves de API: Crie, visualize e exclua as API Keys que suas aplicações usarão para se comunicar com o Neuralogins. Cada chave é única para uma aplicação e deve ser mantida em segredo no backend.
  • Visualizar Histórico de Eventos: Acompanhe os eventos de login e outras atividades relacionadas aos usuários finais das suas aplicações.
  • Gerenciar Usuários Finais: (Seção futura) Em breve, você poderá visualizar e gerenciar os usuários finais que se autenticam através das suas aplicações.
  • Documentação: Consulte este guia para entender como integrar o Neuralogins nas suas aplicações.

O fluxo geral de integração envolve:

  1. Obter uma API Key: Gere uma API Key única para cada aplicação que usará o Neuralogins.
  2. Integrar sua Aplicação: Use a API Key no backend da sua aplicação para se comunicar com a API Neuralogins, preferencialmente usando nosso SDK oficial.
  3. Autenticar Usuários Finais: Implemente o registro e login de usuários finais na sua aplicação utilizando os métodos fornecidos pela API/SDK.

Gerenciando suas Chaves de API

Sua API Key da Aplicação é essencial para conectar sua aplicação ao Neuralogins. Ela identifica sua aplicação cliente (como seu website ou app mobile) e é usada para autenticar as requisições que sua aplicação faz para nossos serviços.

Você pode gerar e gerenciar suas API Keys diretamente neste painel. Cada API Key é única para uma aplicação e deve ser tratada como um segredo.

Como Obter e Gerenciar API Keys:
  • Vá para a seção "Chaves de API" (no menu lateral).
  • Clique em "Gerar Nova API Key" para criar uma chave para uma nova aplicação.
  • Sua nova API Key será exibida apenas uma vez. Copie-a imediatamente e armazene-a em um local seguro no backend da sua aplicação.
  • Você pode ver uma lista das suas API Keys ativas na mesma seção. Por segurança, apenas os IDs parciais são exibidos.
  • Se uma API Key for comprometida ou não for mais necessária, você pode excluí-la nesta seção.

CRÍTICO: Mantenha sua API Key segura no backend da sua aplicação. Nunca a exponha no código frontend (navegador, app)!

Isolamento de Dados com Project Group Key

Para garantir a segurança e o isolamento dos dados dos seus usuários finais, o Neuralogins utiliza o conceito de Project Group Key. Cada API Key que você gera está associada a um Project Group Key específico.

Isso significa que os usuários finais autenticados através de uma API Key só terão seus dados (como informações de login e dados seguros associados) visíveis e acessíveis dentro do grupo definido por aquela Project Group Key. Se você tiver múltiplas aplicações (com API Keys diferentes), os usuários de uma aplicação não verão os dados dos usuários de outra, mesmo que usem o mesmo email.

Pense no Project Group Key como um contêiner seguro para os dados dos usuários de um conjunto específico de suas aplicações.

Integrando com sua Aplicação (Via SDK)

A maneira recomendada e mais segura de integrar sua aplicação com o Neuralogins é utilizando nossa biblioteca JavaScript oficial: neuralogins-client.

O SDK agora gerencia automaticamente a sua API Key da Aplicação e os tokens de segurança (JWT e Refresh Token), simplificando drasticamente a integração.

Passos para Integração:
  1. Instale o SDK: Se sua aplicação usa Node.js (backend) ou um framework frontend com suporte a módulos (como React, Vue, Angular), instale o pacote via npm: 
    npm install neuralogins-client
  2. Importe e Inicialize: Importe o SDK e inicialize-o com sua API Key da Aplicação e a URL base da sua API. A API Key deve ser carregada de forma segura (ex: variáveis de ambiente no backend). 
    // Importe o cliente
import NeuraloginsClient from 'neuralogins-client'; // Para módulos ES6
// Ou: const NeuraloginsClient = require('neuralogins-client'); // Para CommonJS (Node.js)

// Substitua pela API Key gerada neste painel para esta aplicação específica
const YOUR_APPLICATION_API_KEY = 'SUA_API_KEY_GERADA_AQUI'; // Mantenha isso seguro!

// Substitua pela URL base da sua API Neuralogins
const NEURALOGINS_API_URL = 'http://localhost:5000/api'; // Exemplo

// Inicialize o cliente
const nlClient = new NeuraloginsClient(YOUR_APPLICATION_API_KEY, NEURALOGINS_API_URL);

// Agora você pode usar a instância 'nlClient' para chamar os métodos de autenticação e gerenciamento de usuários finais.
  3. Use os Métodos do SDK (Autenticação Automática): Utilize os métodos fornecidos pelo objeto nlClient. O SDK cuidará de enviar a API Key (para rotas públicas) e o token JWT (para rotas protegidas) automaticamente. Ele também gerencia o Refresh Token para renovar a sessão quando necessário.
    Exemplos de Uso:
    Registro de Usuário: 
    try {
  const userData = await nlClient.registerEndUser('Nome do Usuário', 'email@example.com', 'senhaSegura123');
  console.log('Usuário registrado:', userData);
} catch (error) {
  console.error('Erro no registro:', error);
}
    Login de Usuário: 
    try {
  const loginData = await nlClient.loginEndUser('email@example.com', 'senhaSegura123');
  console.log('Login bem-sucedido:', loginData);
  // A SDK armazena os tokens internamente após o login
} catch (error) {
  console.error('Erro no login:', error);
}
    Acessando Rota Protegida:

    Após o login, simplesmente chame os métodos da SDK para acessar endpoints protegidos. A SDK adicionará o cabeçalho de autorização e gerenciará o refresh de token automaticamente.

    try {
  const secureData = await nlClient.getSecureData(); // Exemplo de chamada a rota protegida
  console.log('Dados seguros:', secureData);
  // Se o token expirar, a SDK tentará renová-lo e repetir a requisição
} catch (error) {
  console.error('Erro ao obter dados seguros:', error);
  // Erros de autenticação (após falha no refresh) serão lançados aqui
}
    Logout:

    Para fazer logout, chame o método logout. Isso limpará os tokens locais e notificará o servidor para invalidar o Refresh Token.

    try {
  await nlClient.logout();
  console.log('Logout bem-sucedido.');
} catch (error) {
  console.error('Erro no logout:', error);
}

    Consulte o README completo do SDK `neuralogins-client` para a lista completa de métodos e mais detalhes.

Lembre-se: A apiKey deve ser carregada de forma segura e a SDK gerencia os tokens automaticamente. Você não precisa manipular os tokens diretamente na sua aplicação cliente ao usar a SDK.

Considerações de Segurança

  • HTTPS Obrigatório: Sempre use HTTPS para comunicação entre sua aplicação e a API Neuralogins em produção para proteger senhas e a API Key.
  • Proteja sua API Key: A apiKey da aplicação é um segredo. Armazene-a de forma segura no backend da sua aplicação (ex: variáveis de ambiente) e nunca a exponha no código do lado do cliente.
  • Senhas Fortes: Incentive ou force seus usuários finais a usarem senhas fortes.
  • Validação de Token JWT: Após o login, sua aplicação receberá um token JWT. Seu backend deve validar este token (verificar assinatura e expiração) antes de confiar nele para autorizar acesso a recursos protegidos. O SDK `neuralogins-client` oferece uma função para isso.
  • Rate Limiting e Brute Force: A API Neuralogins possui proteções básicas, mas considere implementar suas próprias camadas de segurança na sua aplicação cliente, se necessário.

Gerenciamento de Usuários Finais

Esta seção detalha como gerenciar usuários finais da sua aplicação através da API.

Registro de Usuário Final

Registra um novo usuário final para a sua aplicação. Requer a API Key da Aplicação no cabeçalho x-api-key.

curl -X POST \
  http://localhost:3000/api/client/register \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: <SUA_API_KEY>' \
  -d '{
    "email": "<EMAIL_DO_USUARIO>",
    "password": "<SENHA_DO_USUARIO>"
}'
Login de Usuário Final

Autentica um usuário final existente. Requer a API Key da Aplicação no cabeçalho x-api-key. A resposta bem-sucedida inclui um token JWT no corpo da resposta.

curl -X POST \
  http://localhost:3000/api/client/login \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: <SUA_API_KEY>' \
  -d '{
    "email": "<EMAIL_DO_USUARIO>",
    "password": "<SENHA_DO_USUARIO>"
}'
Exclusão de Usuário Final

Exclui um usuário final específico. Requer a API Key da Aplicação no cabeçalho x-api-key e o ID do Usuário na URL.

curl -X DELETE \
  http://localhost:3000/api/client/user/<ID_DO_USUARIO> \
  -H 'x-api-key: <SUA_API_KEY>'
Solicitação de Redefinição de Senha

Inicia o processo de redefinição de senha para um usuário final. Requer a API Key da Aplicação no cabeçalho x-api-key e o email do usuário no corpo da requisição.

curl -X POST \
  http://localhost:3000/api/client/forgot-password \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: <SUA_API_KEY>' \
  -d '{
    "email": "<EMAIL_DO_USUARIO>"
}'
Confirmação de Redefinição de Senha

Confirma a redefinição de senha usando o token recebido por email. Requer o token de redefinição e a nova senha no corpo da requisição.

curl -X POST \
  http://localhost:3000/api/client/reset-password \
  -H 'Content-Type: application/json' \
  -d '{
    "token": "<TOKEN_DE_REDEFINICAO>",
    "newPassword": "<NOVA_SENHA>"
}'
